使用上設定防火牆陷阱

最近公司AP Server 程式使用 mail 發信的時候
明明有設定防火牆開通 IP
結果不能發出去

後來發現是 AP Server IP 是吃別的 IP
這邊 mail server 是不同網段，所以會走 gateway
IP Gateway 預設不是走 interface AP 這邊
可以用 ip route show
主要是看這一行

default via 192.168.1.1 dev eth0 src 192.168.1.202 metric 202
